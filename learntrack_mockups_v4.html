Employee Portal — Flow Canvas
LearnTrack
🎓 Employee Portal — Full Flow
Login → Dashboard → My Levels → Submissions → Feedback History
🔐 Login — Employee
  • Select "Employee" tab · enter email + password
  • JWT token issued · redirected to dashboard
↓
📊 Dashboard
  • 4 cards: Levels Completed, Current Level, Review Pending, Rework Required
  • L1/L2/L3 progress bars with status badges
  • Recent activity feed
🎓 My Levels
  • L1 card: completed — shows manager name + feedback
  • L2 card: review pending — shows "Waiting for review from Manager: Rajesh Kumar"
  • L3: locked until L2 completed
📁 Submissions — 3 sub-tabs
📤 Project Submission
  • Project Title
  • Project Description
  • GitHub URL
  • Manager Name dropdown → sets reviewer_id
  • Submit button
📋 Submission History
  • All versions submitted (v1, v2…)
  • Status per version
  • Manager name on each
  • Resubmit form for rework items
💬 Feedback History
  • All manager feedback received
  • Strengths + Improvements per attempt
  • Overall comment
  • Outcome badge (Rework / Completed)
Status Lifecycle — No IN_REVIEW state
NOT_ENROLLED→ENROLLED→IN_PROGRESS→ submit →REVIEW_PENDING→ manager reviews →COMPLETED
REVIEW_PENDING→ rework feedback →REWORK_REQUIRED→ resubmit →REVIEW_PENDING(repeats)

Screen 14
Manager Portal — Flow Canvas
LearnTrack
👔 Manager Portal — Full Flow
Login → Dashboard → Pending Reviews → Open → Feedback → Completed Reviews
🔐 Login — Manager
  • Select "Manager" tab · role = MANAGER in JWT
  • Redirected to Manager Dashboard on login
↓
📊 Dashboard
  • 4 cards: Pending Reviews, Completed Today, Rework Sent, Total Reviewed
  • Recent activity: new review requests, completions, reworks
📋 Reviews — 2 sub-tabs
  • Pending Reviews: employees who chose this manager · form per card
  • Completed Reviews: full history of approved + rework outcomes
Review Flow — Employee selects manager, no auto-assignment
1. Employee submits project
→
2. Employee selects manager name
→
3. Manager receives review request
4. Manager opens GitHub repository
→
5. Manager fills strengths + improvements + overall feedback
→
6. Decision
✅ Approve →COMPLETED | ↩ Request Rework →REWORK_REQUIRED→ employee resubmits →REVIEW_PENDING
Statuses: NOT_ENROLLED · ENROLLED · IN_PROGRESS · REVIEW_PENDING · REWORK_REQUIRED · COMPLETED  —  No IN_REVIEW state.
