EMPLOYEE PORTAL — APPLICATION FLOW CANVAS
Learning Tracker · ProductSquads
🎓 Employee Portal — Complete Application Flow
Full navigation flow — from login to level completion
🔐 Login / Register
  • Email + password authentication
  • Register with name, employee ID
  • Role: Employee (default)
↓
📊 Dashboard
  • Total enrolled levels count
  • In-progress & completed levels
  • L1/L2/L3 progress bars
  • Quick action buttons linking to My Learning & Submissions
↓
🎓 My Learning
  • View all 3 levels (L1/L2/L3) with status badges
  • Enroll Now → Start Learning → in_progress
  • Submit Project button (redirects to Submissions)
  • "View Review Status" opens review history modal
  • Level card shows GitHub repo link when submitted
📁 Submissions
  • Active submissions list (in_progress, review_pending, rework_required)
  • Submit GitHub URL form for in_progress levels
  • Inline review history per submission card
  • "Resubmit" button + form when status = rework_required
  • Status automatically updated after submit/resubmit
Enrollment Status Lifecycle
not enrolled→enrolled→in_progress→ submit →review_pending→ approve →completed
review_pending→ rework →rework_required→ resubmit →review_pending(repeats until approved)

ADMIN / MANAGER PORTAL — APPLICATION FLOW CANVAS
Learning Tracker · ProductSquads
👥 Admin / Manager Portal — Complete Flow
Login → Admin Dashboard → Employees / Reviews / Reports
🔐 Login / Register
  • Email + password authentication
  • Register with "Is Admin" checkbox
  • Role: admin — full system access
↓
📊 Dashboard
  • 6 stat cards: total employees, L1/L2/L3 completed, pending, rework
  • Recent enrollment changes feed
  • Quick navigation to all sections
↓
📊 Dashboard
  • Overview stats with 6 live stat cards
  • Recent activity feed with status badges
  • Click-through to employee details
👥 Employees
  • Search by name, email, employee ID
  • Per-employee L1/L2/L3 status grid
  • Detail modal showing submission links
  • Overall progress status badge
📋 Reviews
  • Pending queue with GitHub repo links
  • Approve (→ completed) or Request Rework
  • Feedback text box for rework requests
  • Completed reviews history section
📈 Reports
  • Bar chart: status breakdown per level
  • Level completion detail cards (L1/L2/L3)
  • Completion rate percentage
Manager Review Decision Flow
submission arrives→review_pending→ Manager reviews →Approve ✅→completed
review_pending→ Request Rework + feedback →rework_required→ Employee resubmits →review_pending
